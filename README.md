# 两仪 (LiangYi) 微服务生态矩阵

### 1. 核心理念
**“易有太极，是生两仪，两仪生四象，四象生八卦。”**
**两仪 (LiangYi)** 框架是一套面向高并发、高可用场景的自研微服务中间件矩阵。它将中国古代哲学中的“阵法”思维引入软件架构，旨在通过八个核心组件（对应八卦位）的协同运作，在瞬息万变的分散式流量中，构建出一套稳如磐石、灵活应变的运行秩序。

* **两仪：** 代表框架支持 **Java (刚)** 与 **Go (柔)** 双栈实现，刚柔并济，互通有无。
* **阵法：** 每一个中间件不再是孤立的工具，而是阵法中的一个方位，共同守卫系统的稳定性。

---

### 2. 阵法布局 (组件定义)

本项目将微服务治理的各项核心功能映射至八卦方位，赋予其独特的职能内涵：

| 方位 | 组件名称 | 职能定义 (Middleware Role) | 哲学寓意 |
| :--- | :--- | :--- | :--- |
| **乾 (天)** | **Qian-Directory** | **服务注册中心** | 乾为天，为首领。统领全局服务的元数据，是所有节点定位的准则。 |
| **坤 (地)** | **Kun-Profile** | **分布式配置中心** | 坤为地，为载体。承载万物生长的基石，为所有服务提供环境参数与运行配置。 |
| **离 (火)** | **Li-Trace** | **全链路追踪系统** | 离为火，为光明。照亮系统内部黑暗的调用链路，使请求轨迹明察秋毫。 |
| **巽 (风)** | **Xun-Pilot** | **服务治理与负载中心** | 巽为风，为无孔不入。调度流量如清风过岗，灵活实现动态路由与灰度发布。 |
| **艮 (山)** | **Gen-Brake** | **分布式限流与熔断** | 艮为山，为阻挡。在洪峰过载时稳如泰山，强力拦截异常流量，保护核心。 |
| **震 (雷)** | **Zhen-Ticker** | **分布式定时任务** | 震为雷，为动能。定点爆发，准时触达，驱动周期性业务逻辑的自动执行。 |
| **坎 (水)** | **Kan-Vault** | **分布式锁管理器** | 坎为水，为陷也。将并发请求引入有序的竞争“陷阱”，确保资源的排他性访问。 |
| **兑 (泽)** | **Dui-Portal** | **微服务智能网关** | 兑为泽，为开口。作为系统对外的门户，负责协议转换、鉴权与流量汇聚。 |

* **中枢 (消息队列)：** **TaiJi-Flow (太极流)** —— 位于阵法核心，负责各组件间异步数据的阴阳流转与能量解耦。

---

### 3. 技术愿景

* **阵地协同 (Deep Integration)：** 区别于拼凑式的开源组件，**两仪** 矩阵下的 SDK 深度打通。例如：`Gen-Brake` 的限流规则可直接由 `Kun-Profile` 动态下发。
* **双栈共生 (Cross-Language)：** * **LiangYi-J (Java 框架)：** 追求极致的扩展性与生态兼容，适用于复杂的业务逻辑中枢。
    * **LiangYi-G (Go 框架)：** 追求极致的响应速度与低内存开销，适用于高性能边缘计算与网关层。
* **变易应敌 (Adaptive Resilience)：** 框架具备自愈能力，当某个方位（服务节点）失效时，阵法自动平移（故障转移），确保全局不乱。
